мд Создан, можно начинать работу с проектом. Git, система контроля версий-это программа, которая помогает отслеживать изменения в программах, текстовых файлах, больших документах, веб-сайтах и т. д., так же помогает работать в команде, помогает сохранять и склеивать труд разных людей.
    Сначала необходимо установить Git для Windows с официального сайта Git, установить надо в c:\Program Files\Git, после установки переити в Bin и запустить файл bach.exe, откроется терминал- консоль.
    С помощью терминала-командной строки всегда можно перейти к домашней директории по команде cd или символу ~ для выполнения Enter, где в программе вы находитесь сейчас можно узнать введя pwd.
    Чтобы вывести содержимое директории ls.
    Сменить директорию cd (имя папки).
    ls с флагом -a выведет расширенный список.
    .-текущая директория.
    ..-родительская директория.
    mkdir- команда для создания директории.
    mkdir -p можно создать целую структуру директорий.
    touch- команда создания файлов. Эти команды создают файлы и папки в текущей рабочей директории.
    cp- копирование файлов. В простом виде cp два параметра, ( что копируем ) и ( куда копируем ).
    mv- перемещение файлов и папок, сначала указываем имя файла который перемешаем, потом куда перемещаем.
    cat- чтение файлов, работает только с текстовыми файлами.
    rm- удаляет файлы из директорий.
    rmdir- удаляет только пустые директории.
    rm -r- удаляет директории вместе с находящимися в ней файлами.
    &&- ампресады, помогает выполнять сразу несколько команд, разделяя их. mkdir second-project&&cd second-project&&touch second-project
    Aвтозаполнение, что бы найти команду достаточно с каких букв она начинается, ввести их и дважды нажать TAB, терминал покажет список всех соответствующих команд и пути к ним.
    / - перемещение к корневой директории.
    git version- выведет текущую версию Git/
    git config- файл настройки.
    git config --global- файл настройки с ключом, сработает везде.
    Регистрируем свое имя и почту: git config --global user.name " Имя"; git config --global user.email своя почта.
    Вывести содержимое файлов командой: git config --list/
    Сделать папку репозиторием.
    git init- сделали репозиторий, начальная ветка называется main или master.
    Разгидить папку если что то пошло не так: rm-rf.git
    git status- проверка состояния репозитория.
    git add с ключом --all подготавливает к сохранению все файлы репозитория.
    git add.- можно добавить в репозиторий текущюю папку со всеми файлами.
    git commit- делаем первый коммит, а с ключом -m присваеваем коммиту сообщение.
    git commit -m 'Мой первый коммит!'.
    git log- посмотреть историю коммитов.
    GitHub- платформа для хранения IT проектов и совместной работы над ними.
    Можно создавать проекты: Приватный- только для вас; Командный- только для членов команды; Публичный- будет виден всем.Можно присоедениться к чужому проекту и работать над ним вместе с другими людми со всего мира.
    Регистрация на GitHab: на главной странице GitHab нажмите зарегистрироваться, в правом верхнем углу.
    Введите адрес электронной почты, придумайте пароль, введите Имя пользователя, все как в Git.
    Платформа спросит хотите ли вы получать рассылку Y да, N нет.
    Git предложит пройти капчу, сделайте это.
    Введите код который пришлют вам на почту.
    Вы успешно зарегистрировались.
    Создаем удаленный репозиторий: заходим по ссылке https://githab.com/имя которое указали при регистрации.
    Переходим по ссылкам на открывшихся страницах, сначала на вкладку репозиторий, затем нажимаем зеленую кнопку (новый), откроется новое окно, назовите его как вам нравиться,например first-project и смело жмите кнопку создать репозиторий, внизу будет надпись, удаленный репозиторий создан.
    ssh- правила обмена данными: это два ключа для безопасной передачи данных, приватный- только для вас, публичный- доступен всем для шифрования данных.
    Генерация ssh ключей.
    ssh-keygen -t ed25519 -C "электронная почта" та которую указали в GitHub, если ошибка, то набираете: ssh-keygen -t rsa -b4096 -C "электронная почта"/
    Проверка генерации ключей: ls -a ~/.ssh
    Привязываем ssh к GitHub.
    Копируем содержимое ключа в буфер обмена для Windows: clip < ~/.ssh/id_ed25519.pub, или clip < ~/.ssh/id_rsa.pub, если clip не сработал, вводим вместо него cat.
    Перейти на страницу GitHub и выбрать пункт настройки, в меню слева нажать SSH and GPG Keys в открывшейся вкладке выбрать новый SSH ключ. В поле Key вставте ваш ключ из буфера обмена и нажмите добавить ключ.
    Проверьте: ssh -T git@github.com
    Cвязать локальный и удаленный доступ: git remote add origin git@github.com:%Имя Акаунта%/first-project.git
    Убедиться, что репозитории связаны: git remote -v
    Синхронизация локального и удаленного репозитория.
    Отправить изменения на удаленный репозиторий: git push -u origin main или master.
