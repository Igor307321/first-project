1.мд Создан, можно начинать работу с проектом. Git, система контроля версий-это программа, которая помогает отслеживать изменения в программах, текстовых файлах, больших документах, веб-сайтах и т. д., так же помогает работать в команде, помогает сохранять и склеивать труд разных людей.
  2.  Сначала необходимо установить Git для Windows с официального сайта Git, установить надо в c:\Program Files\Git, после установки переити в Bin и запустить файл bach.exe, откроется терминал- консоль.
  3.  С помощью терминала-командной строки всегда можно перейти к домашней директории по команде cd или символу ~ для выполнения Enter, где в программе вы находитесь сейчас можно узнать введя pwd.
  4. Чтобы вывести содержимое директории ls.
  5.  Сменить директорию cd (имя папки).
  6.  ls с флагом -a выведет расширенный список.
  7.  .-текущая директория.
  8.  ..-родительская директория.
  9.  mkdir- команда для создания директории.
  10.  mkdir -p можно создать целую структуру директорий.
  11.  touch- команда создания файлов. Эти команды создают файлы и папки в текущей рабочей директории.
  12.  cp- копирование файлов. В простом виде cp два параметра, ( что копируем ) и ( куда копируем ).
  13.  mv- перемещение файлов и папок, сначала указываем имя файла который перемешаем, потом куда перемещаем.
  14.  cat- чтение файлов, работает только с текстовыми файлами.
  15.  rm- удаляет файлы из директорий.
  16.  rmdir- удаляет только пустые директории.
  17.  rm -r- удаляет директории вместе с находящимися в ней файлами.
  18.  &&- ампресады, помогает выполнять сразу несколько команд, разделяя их. mkdir second-project&&cd second-project&&touch second-project
  19.  Aвтозаполнение, что бы найти команду достаточно с каких букв она начинается, ввести их и дважды нажать TAB, терминал покажет список всех соответствующих команд и пути к ним.
  20.  / - перемещение к корневой директории.
  21.  git version- выведет текущую версию Git/
  22.  git config- файл настройки.
  23.  git config --global- файл настройки с ключом, сработает везде.
  24.  Регистрируем свое имя и почту: git config --global user.name " Имя"; git config --global user.email своя почта.
  25.  Вывести содержимое файлов командой: git config --list/
  26.  Сделать папку репозиторием.
  27.  git init- сделали репозиторий, начальная ветка называется main или master.
  28.  Разгидить папку если что то пошло не так: rm-rf.git
  29.  git status- проверка состояния репозитория.
  30.  git add с ключом --all подготавливает к сохранению все файлы репозитория.
  31.  git add.- можно добавить в репозиторий текущюю папку со всеми файлами.
  32.  git commit- делаем первый коммит, а с ключом -m присваеваем коммиту сообщение.
  33.  git commit -m 'Мой первый коммит!'.
  34.  git log- посмотреть историю коммитов.
  35.  GitHub- платформа для хранения IT проектов и совместной работы над ними.
  36.  Можно создавать проекты: Приватный- только для вас; Командный- только для членов команды; Публичный- будет виден всем.Можно присоедениться к чужому проекту и работать над ним вместе с другими людми со всего мира.
  37.  Регистрация на GitHab: на главной странице GitHab нажмите зарегистрироваться, в правом верхнем углу.
  38.  Введите адрес электронной почты, придумайте пароль, введите Имя пользователя, все как в Git.
  39.  Платформа спросит хотите ли вы получать рассылку Y да, N нет.
  40.  Git предложит пройти капчу, сделайте это.
  41.  Введите код который пришлют вам на почту.
  42.  Вы успешно зарегистрировались.
  43.  Создаем удаленный репозиторий: заходим по ссылке https://githab.com/имя которое указали при регистрации.
  44.  Переходим по ссылкам на открывшихся страницах, сначала на вкладку репозиторий, затем нажимаем зеленую кнопку (новый), откроется новое окно, назовите его как вам нравиться,например first-project и смело жмите кнопку создать репозиторий, внизу будет надпись, удаленный репозиторий создан.
  45.  ssh- правила обмена данными: это два ключа для безопасной передачи данных, приватный- только для вас, публичный- доступен всем для шифрования данных.
  46.  Генерация ssh ключей.
  47.  ssh-keygen -t ed25519 -C "электронная почта" та которую указали в GitHub, если ошибка, то набираете: ssh-keygen -t rsa -b4096 -C "электронная почта"/
  48.  Проверка генерации ключей: ls -a ~/.ssh
  49.  Привязываем ssh к GitHub.
  50.  Копируем содержимое ключа в буфер обмена для Windows: clip < ~/.ssh/id_ed25519.pub, или clip < ~/.ssh/id_rsa.pub, если clip не сработал, вводим вместо него cat.
  51.  Перейти на страницу GitHub и выбрать пункт настройки, в меню слева нажать SSH and GPG Keys в открывшейся вкладке выбрать новый SSH ключ. В поле Key вставте ваш ключ из буфера обмена и нажмите добавить ключ.
  52.  Проверьте: ssh -T git@github.com
  53.  Cвязать локальный и удаленный доступ: git remote add origin git@github.com:%Имя Акаунта%/first-project.git
  54.  Убедиться, что репозитории связаны: git remote -v
  55.  Синхронизация локального и удаленного репозитория.
  56.  Отправить изменения на удаленный репозиторий: git push -u origin main или master.
